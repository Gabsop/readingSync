### Completed (continued)
- [x] EPUB parser module (`mobile/lib/epub-parser.ts`)
  - Reads .epub (zip) via expo-file-system `File.arrayBuffer()` + JSZip
  - Parses META-INF/container.xml → locates OPF rootfile
  - Parses OPF → extracts metadata (title, author, cover, language, etc.)
  - Builds manifest map (id → href, mediaType, properties)
  - Extracts spine reading order with resolved hrefs from manifest
  - Parses TOC: EPUB 3 nav document and EPUB 2 NCX fallback
  - Cover image extraction (EPUB 3 properties, EPUB 2 meta, id heuristic)
  - `readChapter()` helper for accessing chapter XHTML by spine index
  - Dependencies added: jszip, fast-xml-parser
- [x] Basic XHTML → React Native component mapping (`mobile/lib/xhtml-renderer.tsx`)
  - Renders EPUB XHTML as native RN components (Text, View, Image)
  - Supports: p, h1–h6, em/i, strong/b, u, s/del, sup/sub, a, span
  - Supports: ul/ol/li lists, blockquote, hr, br, img, basic tables
  - Inline CSS style parsing (font-weight, font-style, text-align, color, font-size, margins)
  - Reader screen wired up: loads book from SQLite, parses EPUB, renders chapters
  - Chapter-by-chapter navigation (prev/next) with loading/error states
- [x] Local SQLite database schema
  - Tables: books, reading_positions, sync_queue, settings
  - WAL journal mode, foreign keys enabled
  - Schema versioning via PRAGMA user_version for future migrations
  - SQLiteProvider wired into root layout with Suspense
  - Database accessible via `useSQLiteContext()` hook from all screens
