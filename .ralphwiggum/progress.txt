### Completed (continued 4)
- [x] API key generation endpoint for KOReader plugin
  - New `web_api_keys` table: userId (FK), prefix, keyHash (SHA-256, unique), name, lastUsedAt, createdAt, revokedAt
  - Drizzle migration generated (`0001_fearless_thor_girl.sql`)
  - `POST /api/auth/api-keys` — generates a new API key (returns full key once, stores SHA-256 hash)
  - `GET /api/auth/api-keys` — lists active keys for the authenticated user (prefix, name, lastUsedAt)
  - `DELETE /api/auth/api-keys` — soft-revokes a key by setting revokedAt
  - `getSessionFromRequest()` now resolves `x-api-key` header: hashes the incoming key, looks up in
    `web_api_keys` (excluding revoked), joins to `web_users`, updates `lastUsedAt` fire-and-forget
  - Key format: `rs_k_<32 random bytes base64url>`, prefix stored as first 13 chars for display
  - Utility module (`src/server/auth/api-key-utils.ts`): `generateApiKey()` and `hashApiKey()`
  - TypeScript passes cleanly

### Completed (continued)
- [x] EPUB parser module (`mobile/lib/epub-parser.ts`)
  - Reads .epub (zip) via expo-file-system `File.arrayBuffer()` + JSZip
  - Parses META-INF/container.xml → locates OPF rootfile
  - Parses OPF → extracts metadata (title, author, cover, language, etc.)
  - Builds manifest map (id → href, mediaType, properties)
  - Extracts spine reading order with resolved hrefs from manifest
  - Parses TOC: EPUB 3 nav document and EPUB 2 NCX fallback
  - Cover image extraction (EPUB 3 properties, EPUB 2 meta, id heuristic)
  - `readChapter()` helper for accessing chapter XHTML by spine index
  - Dependencies added: jszip, fast-xml-parser
- [x] Basic XHTML → React Native component mapping (`mobile/lib/xhtml-renderer.tsx`)
  - Renders EPUB XHTML as native RN components (Text, View, Image)
  - Supports: p, h1–h6, em/i, strong/b, u, s/del, sup/sub, a, span
  - Supports: ul/ol/li lists, blockquote, hr, br, img, basic tables
  - Inline CSS style parsing (font-weight, font-style, text-align, color, font-size, margins)
  - Reader screen wired up: loads book from SQLite, parses EPUB, renders chapters
  - Chapter-by-chapter navigation (prev/next) with loading/error states
- [x] Local SQLite database schema
  - Tables: books, reading_positions, sync_queue, settings
  - WAL journal mode, foreign keys enabled
  - Schema versioning via PRAGMA user_version for future migrations
  - SQLiteProvider wired into root layout with Suspense
  - Database accessible via `useSQLiteContext()` hook from all screens

## M2 — Reading Experience

### Completed
- [x] Pagination engine with accurate text measurement
  - Paginated ScrollView with `snapToInterval` based on available screen height
  - Content height measured via `onContentSizeChange`, page count = ceil(contentHeight / pageHeight)
  - Tap-zone navigation: left 40% = previous page, right 40% = next page (Apple Books pattern)
  - Center 20% reserved for controls overlay toggle (future task)
  - Automatic chapter transitions: last page → next chapter, first page back → previous chapter
  - Global page numbering across chapters (e.g., "15 of 561")
  - Chapter page counts cached in ref for cross-chapter page calculation
  - EPUB parsed once and cached in ref (avoids re-parsing on chapter change)
  - `ParsedEpub` type exported from epub-parser for use in reader screen
  - `PaginatedChapter` reusable component created (`mobile/lib/paginated-view.tsx`)
  - Page indicator in footer (replaces old chapter-level prev/next buttons)
  - TypeScript passes cleanly

### Completed (continued)
- [x] Page turn gestures + slide animation
  - Horizontal pan gesture via `react-native-gesture-handler` Gesture.Pan()
  - Finger-tracked slide animation using `react-native-reanimated` shared values
  - Swipe triggers at 25% of screen width or 500px/s velocity (fast flick)
  - Slide-out animation (250ms, cubic ease-out) before page change
  - Spring snap-back on cancelled swipe (damping 20, stiffness 300)
  - Subtle shadow overlay grows with swipe distance for depth effect
  - Tap zones preserved: left 40% = prev, right 40% = next, center 20% = controls (future)
  - Gesture composition: `Gesture.Race(pan, tap)` — pan wins if horizontal, tap wins otherwise
  - Pan activates after 15px horizontal movement, cancels on >10px vertical (no scroll conflict)
  - Light haptic feedback (`expo-haptics` ImpactFeedbackStyle.Light) on every page turn
  - Animation lock prevents double-triggers during slide-out
  - TypeScript passes cleanly
